# Claude Code CLI 프롬프트 — "하루 한 걸음" 재활 지원 앱 (Android 우선, React Native)

## 프로젝트 개요
뇌병변 장애(혈액암 전이로 인한 뇌척수액 침범 후 방사선/항암치료 후유증)로 인지력·기억력이 저하되고 손 힘이 약한 60대 여성 환자를 위한 재활 지원 앱을 만들려고 해.
환자는 현재 재활병원에 입원 중이고, 하체 재활은 병원 물리치료(PT)로 이미 진행 중이라 앱에서 다루지 않아. 앱의 목표는 (1) 인지력·기억력 훈련, (2) 손 힘 훈련 보조, (3) 입원 중 안전/의사소통 지원이야.

- **앱 표시 이름**: 하루 한 걸음 (Haru Han-geoleum)
- **프로젝트 폴더명 / 패키지명**: `haru` (예: `com.haruhangeoleum.haru`)
- **저장소명**: `haru-app`

Expo 기반 React Native + TypeScript로 시작하고, Android부터 만들고 나중에 iOS(TestFlight)로 확장할 계획이야.

## 반드시 지킬 UX 제약 (환자 특성 반영)
- **손에 힘이 없음**: 롱프레스(꾹 누르기), 스와이프/드래그, 더블탭 절대 사용 금지. 모든 동작은 "가벼운 탭 한 번"으로 완결되어야 함.
- **위험한 동작에는 실수 방지 장치**: 탭 → 3초 카운트다운 표시 → 취소 안 하면 실행되는 패턴 사용 (도움 요청 등). 카운트다운 취소도 가벼운 탭 한 번으로.
- **인지 부하 최소화**: 한 화면 = 한 가지 행동. 네비게이션 깊이는 최대 2단계. 버튼 위치·색상·아이콘은 화면마다 절대 바뀌지 않아야 함.
- **큰 터치 영역**: 버튼 최소 높이 52~56px, 글자 크기 최소 16px 이상, 버튼 간 여백 16px 이상.
- **병원 입원 상황 고려**: 소리 나는 알림 금지. 모든 알림은 진동(haptics) + 화면 표시로만. 음성 인터페이스(STT/TTS) 사용하지 않음 — 병실에 다른 환자가 있어 말하기 어려운 환경.
- **화면은 거치대에 세워두고 씀**: 화면 자동 잠금 시간을 길게, 자주 쓰는 버튼은 화면 하단 1/3 안에 배치.

## 기술 스택
- Expo (managed workflow) + React Native + TypeScript
- expo-haptics (진동), expo-notifications (알림), expo-image-picker (가족 사진 등록)
- 상태 관리: Zustand 또는 Context API (가벼운 걸 추천, 이유 설명해줘)
- 로컬 데이터 저장: expo-sqlite 또는 AsyncStorage (환자 개인 훈련 기록·문제은행용, 추천 방식 제안해줘)
- 네비게이션: React Navigation
- **백엔드: Firebase** (직접 서버 구축 안 함)
  - Firestore: 도움 요청, 복약 기록, 훈련 진행 기록을 보호자 앱/대시보드와 동기화
  - Cloud Messaging(FCM): 도움 요청 발생 시 보호자 폰에 실시간 푸시 알림
  - 환자 개인 훈련 데이터(문제 풀이 기록 등)는 로컬에 우선 저장하고, 보호자가 봐야 하는 요약 데이터만 Firestore로 동기화하는 구조로 설계해줘 (전체를 다 서버로 안 올려도 됨)

## 폴더 구조 (이 구조로 시작해줘)
```
src/
  screens/
    Home/                 // 오늘 안내 + 훈련 메뉴 진입점
    CognitiveTraining/    // 지남력·주의력 훈련
    MemoryTraining/       // 가족 회상, 일화기억, 단기기억 등
    HandExercise/         // 손 힘 훈련 (영상 안내 + 횟수 카운터)
    HelpRequest/          // 도움 요청 (탭 → 3초 카운트다운 → 전송)
    Medication/           // 복약 알림 (진동 기반)
    CaregiverDashboard/   // 보호자용 진행 기록 화면
  components/             // 공통 큰 버튼, 카드 등 재사용 컴포넌트
  services/               // 알림, 진동, 로컬 저장소 로직
  data/                   // 문제은행(정적 데이터) + 랜덤 선택 알고리즘
  types/                  // TypeScript 타입 정의
```

## 인지훈련 / 기억력 훈련 콘텐츠 구성
문제 유형을 다양하게 준비하고, 아래 규칙에 따라 "규칙 있는 랜덤"으로 매일 문제를 뽑아줘.

**인지훈련 (지남력·주의력)**
- 오늘 날짜/요일/계절 맞히기
- 지금이 오전인지 오후인지 맞히기
- 지금 있는 장소(병원 이름, 병실) 맞히기
- 같은 그림 짝 찾기 (카드 매칭)
- 정해진 색깔/모양만 순서대로 터치하기
- 다른 그림 찾기

**기억력 훈련 (일화기억·사람·단기기억)**
- 오늘 아침 뭘 먹었는지 회상
- 어제 있었던 일 회상 퀴즈 (보호자가 전날 입력해두는 내용 기반)
- 가족 사진 보고 이름 맞히기
- 담당 의료진 이름 회상
- 단어 3~5개 잠깐 보여준 뒤 나중에 골라내기
- 사진 여러 장을 시간 순서대로 배열하기

**랜덤 선택 알고리즘 규칙**
- 하루에 인지훈련 2개, 기억력훈련 2개씩 고정 개수로 노출
- 최근 2~3일 안에 나왔던 유형은 오늘 뽑기에서 제외 (다양성 보장)
- 유형별 정답률을 기록해두고, 정답률이 낮은 유형일수록 뽑힐 가중치를 높임 (약한 영역 보강)
- 같은 날 연속으로 어려운 유형만 나오지 않도록, 쉬운 유형 최소 1개는 섞어서 배치
- "다른 문제로 바꿔주세요" 버튼으로 오늘 문제를 다시 뽑을 수 있게 함
- 이 로직은 `data/exerciseSelector.ts` 같은 파일로 분리해서 테스트 가능하게 짜줘

## 사용법 안내
아래는 단계별로 나눠진 프롬프트야. **한 번에 하나씩** Claude Code CLI에 붙여넣어줘. 이전 단계 결과물이 이미 프로젝트에 반영되어 있다는 걸 전제로 다음 단계를 진행해줘. 각 단계 시작할 때 이전 단계에서 만든 컴포넌트(`BigButton` 등)와 폴더 구조를 그대로 재사용해줘.

### 각 단계 완료 기준 (필수, 모든 단계 공통)
각 단계 구현이 끝났다고 보고하기 전에 반드시 아래를 지켜줘.
1. 구현이 끝나면 그 자리에서 실제로 테스트해줘 (타입체크 `npx tsc --noEmit`, 관련 유닛/로직 테스트, 가능하면 에뮬레이터·실기기 동작 확인까지). 테스트 없이 "완료했다"고 말하지 마.
2. 오류가 나오면 겉으로 보이는 증상만 고치지 말고 원인을 먼저 파악한 뒤 수정하고, 수정 후 다시 테스트해서 오류가 사라진 걸 확인할 때까지 이 과정을 반복해줘.
3. 이 프로젝트는 여러 PC를 오가며 작업해. 그러니 지금 이 컴퓨터에만 있는 파일·캐시·전역 설정에 의존하는 코드를 만들지 말고, 재현에 필요한 파일(`package-lock.json` 등)은 항상 커밋해줘. 새 환경변수나 API 키가 필요하면 실제 값 대신 `.env.example`처럼 키 이름만 저장소에 남기고, 어떤 값을 채워야 하는지·어디서 발급받는지 알려줘.
4. 가능하면 `git clone` 직후 상태(즉 다른 PC에서 막 받은 상태)를 가정하고 `npm install` → 타입체크/빌드가 깨끗하게 되는지 한 번 더 점검해줘.

### 참고해야 할 것
- **Firestore 보안 규칙**: Firebase Auth를 아직 연동하지 않아서, `helpRequests` / `medicationSummaries` / `appActivity` / `memoryPrompts` / `caregiverDevices` / `exerciseAccuracySummary` / `handExerciseSummaries` 컬렉션에 인증 없이 읽고 쓸 수 있도록 콘솔에서 규칙을 열어둬야 지금 코드가 동작함. 실제 서비스 전에는 Auth 붙이고 규칙을 좁혀야 함.

- **실기기 푸시 알림 테스트**는 Expo Go가 아니라 커스텀 dev client 빌드가 필요함 (SDK 53부터 Expo Go는 원격 푸시 미지원). `eas build --profile development` + `eas credentials`로 FCM 자격증명 연결은 사용자 계정으로 직접 해야 함.

## 단계 1 — 프로젝트 스캐폴딩 + 복약 알림 화면 ✅ 완료
가장 먼저 `Medication` 화면을 만들어줘. 요구사항:
- 지정 시간에 진동으로만 알림 (소리 없음), 화면 전체가 약 이름 + 아이콘으로 전환되는 형태
- 화면에는 큰 버튼 2개만: "먹었어요" / "간호사가 아직 안 주셨어요"
- 버튼은 탭 한 번으로 즉시 처리 (롱프레스/카운트다운 없음 — 복약 확인은 즉시 처리가 맞음)
- 복약 기록은 로컬에 저장하고, 추후 보호자 대시보드에서 조회 가능하도록 데이터 구조 설계

**요청사항**
1. `haru`라는 이름으로 위 폴더 구조로 Expo 프로젝트 스캐폴딩 해줘 (`npx create-expo-app` TypeScript 템플릿 기준)
2. 공통 컴포넌트로 `BigButton` (최소 크기/여백 규격 반영한 재사용 버튼)부터 만들어줘
3. 그 다음 `Medication` 화면과 로컬 저장 로직을 구현해줘
4. 상태관리/저장소 라이브러리 선택 시 왜 그걸 골랐는지 간단히 설명해줘
5. 코드에 UX 제약(롱프레스 금지, 진동 기반 알림 등)이 지켜지도록 주석으로 이유를 남겨줘
6. Firebase는 이번 단계에서는 연동하지 말고, 나중에 `HelpRequest`와 `CaregiverDashboard` 화면 만들 때 연동할 수 있도록 `services/firebase.ts` 자리만 비워서 구조만 잡아줘
7. 인지훈련/기억력훈련의 랜덤 선택 알고리즘(`data/exerciseSelector.ts`)은 이번 단계에서는 만들지 말고, 함수 시그니처와 타입 정의만 미리 잡아줘 (다음 단계에서 본격 구현 예정)

## 단계 2 — Firebase 연동 + 도움 요청(HelpRequest) 화면 ✅ 완료
1단계에서 비워둔 `services/firebase.ts`를 실제로 구현하고, `HelpRequest` 화면을 만들어줘.

**요구사항**
- Firebase 프로젝트 연동 (Firestore + Cloud Messaging). 설정 파일은 `.env`로 분리해서 키가 코드에 노출되지 않게 해줘
- `HelpRequest` 화면 흐름: 화면 대부분을 차지하는 원형 버튼 → 가볍게 탭 한 번 → 3초 카운트다운 화면(취소 버튼도 탭 한 번으로 동작) → 카운트다운 끝나면 Firestore에 이벤트 기록 + 보호자 폰에 FCM 푸시 알림 전송
- 전송되는 데이터에는 시각, (설정된 경우) 병실 정보가 포함되어야 함
- 전송 완료 후에는 "전달했어요" 확인 화면으로 전환
- 이 버튼/카운트다운 상태 전환 로직은 재사용 가능하게 컴포넌트로 분리해줘 (나중에 다른 확인 절차에도 쓸 수 있도록)
- 네트워크 연결이 안 되는 상황(병원 와이파이 불안정 등)을 대비해서, 전송 실패 시 자동 재시도 로직과 사용자에게 "전송 중" 상태를 보여주는 처리도 넣어줘

## 단계 3 — 보호자 대시보드(CaregiverDashboard) ✅ 완료
`CaregiverDashboard` 화면을 만들어줘. 환자용 화면과는 명확히 분리된 진입점(별도 로그인 또는 역할 구분)으로 설계해줘.

**요구사항**
- 오늘의 복약 이행률(단계 1에서 만든 로컬 기록 중 Firestore로 동기화된 요약 데이터 기준) 표시
- 최근 도움 요청 이력 목록 (시각, 상태)
- 최근 활동 시각(마지막으로 앱을 사용한 시각)
- 추후 단계에서 만들 인지훈련/기억력훈련 결과도 같은 대시보드에 얹을 수 있도록 컴포넌트를 확장 가능한 구조로 짜줘 (카드형 위젯 리스트 패턴 추천)

## 단계 4 — 문제은행 데이터 구조 + 랜덤 선택 알고리즘 ✅ 완료
1단계에서 시그니처만 잡아둔 `data/exerciseSelector.ts`를 본격적으로 구현해줘.

**요구사항**
- 문제 유형: 위 "인지훈련 / 기억력 훈련 콘텐츠 구성" 섹션에 나열된 유형 전체를 타입으로 정의
- 선택 규칙:
  - 하루에 인지훈련 2개, 기억력훈련 2개씩 고정 개수로 노출
  - 최근 2~3일 안에 나왔던 유형은 오늘 뽑기에서 제외
  - 유형별 정답률 기록을 바탕으로, 정답률이 낮은 유형일수록 뽑힐 가중치를 높임
  - 같은 날 연속으로 어려운 유형만 나오지 않도록 쉬운 유형 최소 1개는 포함
- 이 로직은 순수 함수로 짜서 단위 테스트 가능하게 해주고, 실제로 Jest 테스트 코드도 몇 개 같이 작성해줘 (규칙이 의도대로 동작하는지 검증)
- "다른 문제로 바꿔주세요" 요청 시 재선택하는 함수도 별도로 분리해줘

## 단계 5 — 인지훈련 / 기억력훈련 화면 ✅ 완료

**요구사항**

## 단계 6 — 손 힘 훈련(HandExercise) 화면 ← 다음 단계

- 이 화면에 들어가는 운동 종류/횟수는 보호자 대시보드에서 담당 치료사 확인 후 조정할 수 있도록 설정 가능하게 해줘 (하드코딩 금지)
